# Frequently Asked Questions

## GitHub Basics

**What is GitHub?**  
Platform for version control and collaboration on code/documents.

**Do I need to know Git?**  
Basic understanding helps. Start with [GitHub Hello World](https://docs.github.com/en/get-started/start-your-journey/hello-world).

**What can I store?**  
Code, documentation, configuration files. **NOT patient data or PHI/PII.**

## Access & Permissions

**How long for access approval?**  
Usually 1-2 business days after manager approval.

**Can I use personal GitHub?**  
No. Use institutional email and organization account only.

**Why is 2FA required?**  
Security requirement for all users. No exceptions.

## Working with Repositories

**Public or private?**  
Default to private. Public only with approval and no sensitive data.

**Who can see my code?**  
Private repos: Only team members. Public repos: Anyone on internet.

**How often to commit?**  
Commit meaningful changes with clear messages. Quality over quantity.

## Data & Compliance

**What's data classification?**  
Rating system (A-E) for data sensitivity. All repos need `DATA_CLASSIFICATION.md`.

**Can I store research data?**  
Small datasets (<100MB) only. Large data belongs in proper data storage.

**What about backups?**  
GitHub is not a backup solution. Have separate backup strategy.
